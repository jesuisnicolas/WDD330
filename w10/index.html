Client-side From Validation

When we have a form we have to control the information to check it is in the right format. Using client-side validation is an important feature of the user experience, but you also need to perform validation checks in the server side, mainly because client-side validation can be bypassed by a malicious user.

Why do we validate forms?

  • We want to get the richt data in the right format.
  • We want to protect our user's data (I.E: forcing users to choose a secure password)
  • We want to protect ourselves (Users can use forms to damage our application)

There are two types of validation:
Built-in form validation: Uses HTML5 features and has better performance than Javascript validation, but it's not as customizable as Javascript.
Javascript form validation: It's completely customizable, but you have to create it all, or use a library

Built-in Validation:

  • required
  • minlength and maxlength minimum and maximum length for strings.
  • min and max minimum and maximum values for numbers
  • type specifies the type of data (number, date, string, email, etc)
  • pattern specifies the pattern the data have to follow using regular expressions.

If the input is valid, the element matches the :valid CSS pseudo class, and the form will be submitted.

If the input is not valid, the element matches the :invalid CSS pseudo-class and the browser will block the form from being sent.

Javascript Validation

Constraint Validation API: Most browsers support the Constraint Validation API, which consists of methods and properties available on the following elements:

  • HTMLButtonElement
  • HTMLFieldSetElement
  • HTMLInputElement
  • HTMLOutputElement
  • HTMLSelectElement
  • HTMLTextAreaElement

The API makes the following properties available for these elements

validationMessage Returns a message describing the validation constraints that the control doesn't satisfy

validity Returns a ValidityState object that contains properties describing the validity of the element. This can be: patternMismatch, tooLong, tooShort, rangeOverflow, rangeUnderflow, typeMismatch, valid, valueMissing.

willValidate Returns true if the elements will be validated when the form is submitted.

Error messages can be customized using Javascript and the validity value, using an event listener in the input, and checking the value of validity.

Using the novalidate attribute in our forms (in the HTML file) we can take complete control of the way the browser displays the errors using js and CSS.

Using Fetch

Fetch is a Javascript interface for accessing and manipulating parts of the HTTP pipeline, such as requests and responses. It also provides a fetch() method that provides an easy, logical way to fetch resources asynchronously.

Fetch can be used by Service Workers.

A basic Fetch:

fetch('http://example.com/movies.json') this takes one argument (path)
.then(response => response.json()) returns a promise containing a response
.then(data => console.log(data)); we show the data in the console.

The fetch() method can accept a second parameter, an init object that allows us to control a number of options.

Fetch can also be used to upload data, like a JSON file or any other kind of file.
