The Window Object

Every JS enviroment has a GLOBAL OBJECT. Any global variable is a property of this object and every function is a method of it. In a browser enviroment, the global object is the window object, which represents the browser's window.

The Browser Object Model

The BOM is a collection of properties and methods that contain information about the browser and computer screen, such as which browser, screen resolution, or pages has been visited before. Every browser window, tab, popup, frame, and iframe has a window object.

If you create a variable without using var, let, or const (I.E. x = 6;) you can access it using window.x; because that variable is part of the window object.

Browser Information

window.navigator.userAgent will return the browser and OS used. (This is probably what some shady pages use to show you "personal" information to sell youa VPN account)

window.location will return information about the URL.

Browser Histoy

window.history have information about previously visited pages in the current browser session.

Opening Windows

You can open a new window using window.open(). It takes 3 parameter: the URL, the title, and a list of attributes.

window.open("http://www.somewebsite.com", "Some Website", "width=400, height=400m resizable=yes");

Screen Information

window.screen() has information about the screen the browser is displayed in.

Cookies

Cookies are name-value pairs separated by semicolons. Cookies were invented so teh browser can "remember" things between sessions. Cookies can be read only by the same domain that created them, with a limit of 20 per domain, and 4KB per cookie.

Cookies are used for Personalization of the webpage, storing preferences, authenticating, and tracking. Cookies are passed between the server and client with each request.

To set a cookie: document.cookie = "name: superman";

Cookies can be given an expiration date with the property expires=date

Adding the string "secure" to the end of the cookie will ensure that it gets transmitted only over an HTTPS connection.

To remove a cookie you need to set the expiration date to any date in the past.

Timing Functions

setTimeout() accepts a callback to a function as first parameter, and a umber of milliseconds as second parameter.

setInterval() accepts a function and a number of milliseconds as its parameters. I will call the provided functions at intervals.

HTML 5 APIs

The data- attribute is a way of embedding data in a webpage using custom attributes that are ignored by the browser. They're provate to the page and can only be used by a JS program.

To create a data attribute: <div id="hero" data-powers="flight superSpeed"> Superman </div>

To access this data attribute: const powers = superman.dataset.powers; Each property can be accessed and modified in this way.

HTML5 Web Storage

Provides a key-value store on the client's computer. It's perfect for storing information about users, and application-specific information that can be used during future sessions.

Differences with cookies:

  • Information stored is not shared with the server on every request.
  • Information is available in multiple windows of the browser (but only if the domain is the same).
  • Storage capacity limit is much larger than the 4KB of cookies.
  • Any data stored does not automatically expire as it does with cookies. This potentially makes cookies a better choice for something like showing a popup once a day.

localStorage.setItem("name", "Walter White");

localStorage.getItem("name"); This will return "Walter White".

Geolocation

Used to determine the geographic position of the device.

navigator.geolocation.getCurrentPosition(youAreHere);

function youAreHere(position) {
console.log(`Latitude: ${position.coords.latitude}, Longitude: ${position.coords.longitude}`);
}

Web Workers

Web Workers allow processes to be running in the background, adding support for concurrency in Javascript. The idea is that any process that will take a long time is carried out on the background, so a website can function without an error message appearing.

To create a new Worker: const worker = new Worker("task.js"); This function takes the name of another javascript file as an argument. If this file exists, it will be downloaded asynchronously. The worker will only start once the file has finished downloading completely. Web workers use the concept of messages to communicate back and forth with the main script.

Websockets

Until now, the main method of communication with the server has been HTTP requests. The problem with HTTP requests is that for each request we get only one response in the moment we make the request. So, what happens if you want/have to wait for the response like in a, lets say, chat application? This was solved partially with AJAX, but there's a protocol called Websockets that help with this.

Websockets allow two-way communication between the server and the client using Push Messages. This means that the connection is kept open and the responses are "pushed" to the client.

Notifications

The notification API alloes you to show notifications using the system's notifications. A benefit of it is that the notifications will be shown even if you're not in the tab that produced the notification.

Before sending notifications you have to ask for permission to the user.

Multimedia

HTML5 introduced the <audio> and <video> tags to deliver multimedia to the user. At the same time, the Audio API allows JS to control the multimedia.

To insert audio: <audio src="/audio.mp3" controls></audio> The 'control' keyword will show the play, pause, stop, etc. controls.

Audio and video fire a number of events (play, pause, volumechange, loadmetadata) that allow us to respond to user's interaction with the multimedia in the page.
