Notes:

Transform > translate, rotate, scale, skew.

Transform doesn't work in inline elements. To make them work you have to add display: inline-block;

Translate: lets you move an element relative to its current position. transform: translate(45px, -45px);

Scaling: allows you to change the dimensions of an element. transform: scale(1.5, 0.25);

Scaled elements will grow or shrink from the center. To change that you can use the transform-origin property.

Rotation: rotates an element around the point of origin by an specified angle value. Positive degrees mean clockwise rotation.

transform: rotate(10deg);

Skew: specifies a skew along the x and y axes. transform: skew(15de, 4deg);

The order in which transforms are written is important because they're applied in that order. If you rotate something and then translate it, it will translate on the rotated axis. This can lead to unexpected behaviors.

Transitions:

CSS transitions are smoother than JS transitions. Transitions allow the values of CSS properties to gradually change over time, providing simple animations.

transition: [property-name] [duration];

Transition Timing: controls the pace of the transition. The options are: ease, linear, ease-in, ease-out, ease-in-out. Another option is to customize the timing using cubic-bezier. Example: transition: all 2s ease-out 5ms;

Animations:

Animations are more powerful than transitions because they allow you to control each step of the animation via keyframes. Transitions have only two keyframes (start and end), but with animations we can define as many keyframes as we want/need. Example:

@keyframes myAnimation { 0% { //properties here } 32% { //more animations here } }

Now that we defined the animation, we can apply it:
.element {
animation: 300ms ease-in 5s forward myAnimation;
}

Canvas API:

The Canvas API gives you the possibility of drawing through Javascript. This improves the performance by avoiding the need to download images. We can draw shapes and lines, arcs and text, gradients, and patterns. Also we can manipulate pixels in images and even video.

SVG:

Scalable Vector Graphics. It preserves the quality at all sizes because it is a vector, not a bitmap. SVGs are "drawn" in an XML file.

SVG graphics are available via the DOM. Each piece of a SVG graphic is his own element, and it can be manipulated using Javascript and CSS.

SVG images can be created in the text editor, or using a Graphics editor, such as Adobe Illustrator.

Canvas vs SVG:

Canvas > immediate mode. Once drawn, the browser "forgets" everything about the canvas, and it can't be modified after its creation.

SVG > retained mode. All information is in memory, so we can access to our graphic and modify it. This consumes more memory, obviously, but it gives us the ability to use our graphic later on.

When to use each?

"Use canvas when you can't use SVG (where can't might mean animating thousands of objects, manipulating each individual pixel in an image, etc). In other words, SVG should be your default choice, canvas your backup plan"

Drag and Drop API:

The Drag and Drop API allows you to specify that some elements are draggable, and then specify what happens when these elements are dragged over or dropped into other elements in the page.

There are two uses for it: 1, Drag files from the computer to the browser in combination with the File API; 2, Drag elements into other elements on the same page.

Steps to use Drag and Drop:

  1. Set the draggable attribute to true on any HTML elemt you want to be draggable.
  2. Add an event listener dor the dragstart event on any draggable element.
  3. Add an event listener for the dragover and drop events on any elements that you want to have accept dropped items.
