Objects:

Objects are like real life entities. They have attributes and functions. Functions represent the user acting over the entity. Functions can also be defined as "self-contained sets of related values and functions (they look like dictionaries with keys and values).

In Javascript, this can be used in any function, even if it's not a method of an object. The value of this is evaluated during runtime, depending on the context. This is interesting, because we can use the same function for diferent objects, and this will always refer to the object which called the function. This is called "unbound this."

Arrow functions don't have this If we try to use it in an arrow function, it will point to the outer function.

Object Literals:

An object can be created directly by wrapping all its properties and methods in curly braces {}. This is a distinguishing feature of JavaScript, because it allows to create an object without a class.

All objects are Mutable.

You can use dot notation or bracket notation to have access to an object properties or methods.

OBJECTS ARE COPIED BY REFERENCE - CAREFUL!!

An object literal can be passed as a parameter to a function. This is useful when you need to pass a large number of parameters, because inside the object the order of the properties doesn't matter.

Namespaces:

To avoid name colissions, we can use "namespaces". A namespace is an object that has functions inside, and it is similar to a "category". It can be use to create different categories of functions to group them inside. This way, I can have two functions with the same name but in different categories, thus, avoiding a collission (because you have to call them using the namespace first). IE: myMath.square() and myDraw.square() are two completely different functions using the same name.

DOM (Document Object Model):

The DOM treats everything on a web page as a node. HTML tags, the text inside, and the attributes of a tag are nodes. The HTML tag is the root node, all the other are child nodes.

Whitespaces are considered text nodes

To get elements:
document.GetElementById('title'). Even when there should be only one element in the whole page for each ID, it’s not enforced by the HTML parser.
getElementsByTagName('li'). Returns all the elements with that tag.
getElementsByClassName('hero').
document.querySelector(). allows you to use CSS notation to find the first element in the document that matches a CSS selector provided as an argument.
document.querySelectorAll(). Is similar, but returns ALL the elements that match.

It's easy to manipulate the attributes using Javascript.

The getAttribute() method returns the value of the attribute provided as an argument.

The setAttribute() method can change the value of an element’s attributes. For example: wonderWoman.setAttribute('class', 'villain');

Aditionally we can use the className() method to change a class.

Useful function to create elements without having to do it manually:
function createElement (tag,text) {
  const el = document.createElement(tag);
  el.textContent = text;
  return el
}

CSS Properties can be changed using Javascript too. For this, you select the element, and use dot notation for the properties.
For Instance (asuming that superman is the selected element): superman.style.border = "red 2px solid";

Events:

Events provide the link between the web page and the user's interactions. Every time the user does something (clicking a link, moving the mouse, or pressing a key, and event occurs, and Javascript can detect it.

Every time the page is clicked, a callback function is called. This way the program continues running while it’s waiting for the click event to happen.

DO NOT USE ONCLICK IN THE HTML. (I've been using it all the time and thought it was great. :P

To use an event listener: addEventListener('click',doSomething); | doSomething is the function that will be fired on click, but DO NOT USE parenthesis.

There are ways to know which node fired the event, and where the mouse was at the moment.

Types of Events:

Mouse events: Click, mouse down, mouse up, doubleclick, moiseover, mouseout, mousemove.

Keyboard events: Keydown, keypress, keyup.

Modifier Keys: Shift, Ctrl, Alt and meta. These keys usually have special functions in the browser or OS, so it is not a good idea to use them.

Touch events: touchstart, touchend, touchmove, touchenter, touchleave, touchcancel.

Event Propagation: when you click an element, you are also clicking all the elements nested inside up in the tree. There are two types of propagation: Bubbling and Capturing. Bubbling works from the bottom to the top, and capture works from the top to the bottom.
